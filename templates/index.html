🛡️ Mini Web Application Firewall – Dashboard Overview

1️⃣ Purpose of This Application

This web application is a security demonstration platform designed to explain and showcase how common web attacks are detected and blocked using Web Application Firewall (WAF) techniques.

This system is not intended for normal end users, but for security monitoring and educational purposes.

2️⃣ Who Can Access This System?

Only authorized security users (such as administrators or security analysts) can log in to this system.

Because of this:
• All login alerts
• Attack detection logs
• Security events

are visible to every logged-in user, as they represent security personnel, not regular users.

3️⃣ What You See on This Dashboard

After successful login, the dashboard displays:

• Detected attack attempts
• Failed and suspicious login activities
• Types of attacks such as: SQL Injection, Cross-Site Scripting (XSS), Brute-Force Attack, Credential Stuffing.

These activities help demonstrate how attacks occur and how the system responds.

4️⃣ Why All Users See Security Alerts

In this mini project:

• All registered users are treated as security dashboard users
• Therefore, attack information is shared to help understand system behavior

In real-world applications, such data would usually be visible only to administrators.

5️⃣ Real-World Application Value

If these security mechanisms are implemented in real-world web applications:

• They can detect malicious activities
• Prevent unauthorized access
• Improve overall application security

This project demonstrates how basic WAF concepts can be applied to protect modern web applications.

6️⃣ Educational Note

This system is built as an academic mini project to:

• Understand web security attacks
• Visualize how attacks are detected
• Learn how firewall-based protection improves application security
